# 📊 حاسبة معادلة الميزانية - مُتّزِن (Motazin)

> **مُتّزِن** هو تطبيق ويب محاسبي متقدم مبني باستخدام **React** و **TypeScript** و **Firebase**، ومصمم لتسهيل فهم وتطبيق معادلة الميزانية الأساسية (*الأصول = الخصوم + حقوق الملكية*) عبر نظام القيد المزدوج.

---

## 🚀 المميزات الرئيسية (Key Features)

* **⚖️ مطابقة القيد المزدوج التلقائية:** التحقق اللحظي من توازن معادلة الميزانية مع تأثيرات حركية (Confetti 🎉) عند التوازن.
* **📈 رسوم بيانية تفاعلية:** لوحة إحصائيات مالية لعرض توزيع الأصول والإيرادات والمصروفات ومعدل التدفق النقدي.
* **🤖 مساعد محاسبي ذكي (AI Advisor):** محادثة فورية مدعومة بـ Gemini AI لتقديم نصائح مالية وتحليل البيانات المحاسبية للمستخدم بشكل لحظي.
* **📸 ماسح المستندات والفواتير الذكي (OCR):** قراءة الفواتير والإيصالات آلياً واستخراج البيانات منها بفضل Tesseract.js والذكاء الاصطناعي لتسجيلها كعمليات مباشرة.
* **🎯 نظام الميزانيات والتنبيهات الذكية:** تعيين ميزانيات للحسابات المختلفة وتلقي تحذيرات فورية عند استهلاك 85% أو تخطي الميزانية المحددة.
* **💾 النسخ الاحتياطي الفوري (Snapshots):** إمكانية حفظ نسخ احتياطية متعددة للبيانات والعمليات واستعادتها أو حذفها في أي وقت بنوافذ تأكيد مخصصة.
* **🌐 متعدد اللغات (i18n):** دعم كامل لـ 10 لغات عالمية (العربية، الإنجليزية، الفرنسية، الإسبانية، إلخ) مع اتجاهات كتابة متجاوبة (RTL/LTR).
* **💱 دعم العملات المتعددة:** حساب القيم المالية بـ 8 عملات مختلفة (ريال عماني، ريال سعودي، دولار، يورو، إلخ).
* **🔒 حماية وأمان متقدمين:** سياسات أمان صارمة للمحتوى (CSP) وقواعد Firestore محكمة للتحقق من صحة المدخلات ونوعها وحجمها.

## 🛠️ الإعداد والتشغيل المحلي (Setup & Installation)

### المتطلبات الأساسية
* مثبت بيئة عمل [Node.js](https://nodejs.org/) (الإصدار 18 أو أحدث).

### خطوات التشغيل
1. **قم بتحميل التبعيات والمكتبات:**
   ```bash
   npm install
   ```

2. **تهيئة متغيرات البيئة:**
   قم بإنشاء ملف `.env` في المجلد الرئيسي للمشروع بناءً على ملف التهيئة التجريبي `.env.example`:
   ```bash
   cp .env.example .env
   ```
   املأ متغيرات البيئة ببيانات الوصول الخاصة بـ Firebase ومفتاح واجهة Gemini API.

3. **تشغيل خادم التطوير المحلي:**
   ```bash
   npm run dev
   ```
   سيتم تشغيل التطبيق محلياً على المسار: `http://localhost:3000`.

## 🔒 إعداد خدمات Firebase (Firebase Configuration)

يتكامل التطبيق مع خدمات Firebase لتوفير تجربة سحابية كاملة:

### 1. قاعدة البيانات Firestore Database
يجب تهيئة Firestore وتطبيق القواعد الأمنية المرفقة في ملف [firestore.rules](file:///c:/Users/LENOVO/Downloads/حاسبة-معادلة-الميزانية/firestore.rules) لضمان حماية بيانات المستخدمين:
* تمنع القواعد وصول أي مستخدم لبيانات الآخرين (`isOwner` validation).
* تمنع تعديل حقول الإنشاء التاريخية (`createdAt`).
* تمنع عمليات الحقن العشوائي لرسائل الدعم عبر إلزام الرسائل بنموذج حقول صارم ومقيد بطول أحرف معين.

### 2. التخزين السحابي Firebase Storage
يستخدم لتخزين وإرفاق المستندات والفواتير والتحقق منها. يجب تفعيل خدمة Firebase Storage في لوحة التحكم وتعيين مجلد باسم `users/{userId}/documents/` للمستندات المرفقة.

### 3. المصادقة Firebase Authentication
يدعم التطبيق المصادقة عبر حسابات Google. يرجى تفعيل **Google Sign-In Provider** من لوحة تحكم Firebase وتوثيق نطاقات التطبيق المسموح بها (localhost ونطاق الاستضافة الفعلي).

## ⚙️ متغيرات البيئة المتاحة (Environment Variables)

يجب توفير المتغيرات التالية لضمان عمل كافة المزايا بكفاءة:

```ini
# Firebase Config (للربط السحابي والمصادقة)
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_firebase_auth_domain
VITE_FIREBASE_PROJECT_ID=your_firebase_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_firebase_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_firebase_messaging_sender_id
VITE_FIREBASE_APP_ID=your_firebase_app_id

# Gemini API Key (للتحليل المالي الذكي ومسح الفواتير)
GEMINI_API_KEY=your_gemini_api_key
```

## 🧪 الأوامر والبرامج المساعدة (Available Scripts)

* `npm run dev`: تشغيل التطبيق في وضع التطوير المحلي مع تحديث تلقائي (HMR).
* `npm run build`: بناء نسخة الإنتاج المحسنة والجاهزة للنشر الخالي من الأخطاء مع التحقق البرمجي لـ TypeScript.
* `npm run test`: تشغيل اختبارات الوحدة المحاسبية والاختبارات التكاملية للمكونات باستخدام **Vitest**.
* `npm run test:e2e`: تشغيل اختبارات المحاكاة الشاملة لواجهات المستخدم باستخدام **Playwright**.
* `npm run preview`: معاينة النسخة المبنية محلياً قبل نشرها.
